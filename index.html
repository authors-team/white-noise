Differential Privacy

Data scientists, analysts, scientific researchers and data-driven policy makers often need to analyze data that contains sensitive personal information that must remain private.

Commonly-used privacy-preserving techniques have been shown to be flawed resulting in reconstruction of private data or leakage of information in some cases.

Differential Privacy is the only approach that offers strong provable privacy assurances, and formal worst-case tracking of information leakage, even when any adversary is assumed to already possess extensive personal information about targets.

Theory Behind Differential Privacy

Differential privacy is a formal mathematical definition; any algorithm that can be proven to meet this definition then inherits formal mathematical guarantees that an individual’s contribution cannot be identified in any output computation. Broadly, DP uses two approaches to achieve these benefits:

  • A small amount of statistical noise is added to each result, precisely tuned to mask the contributions of any individual.
  • The amount of information revealed from each result is calculated and deducted from an overall privacy budget for the dataset, ensuring that accumulated information released will not compromise individual privacy.

    • Original DP Paper

    These parameters are precisely calibrated by the system to offer strong assurances against reidentification, record reconstruction and membership attacks.

    WhiteNoise

    The WhiteNoise system is designed to be a layer situated between user queries and data systems. When a user queries the data (or trains a model), the WhiteNoise system

    Adds the statistical noise to the resulting queries,
    Calculates the information budget used by the query,
    And subtracts from the remaining budget.

    The system is composed of flexible basic building blocks designed to easily plug into many production data environments. Implementations are based on vetted and mature differential privacy research and practice.

    Native Runtime Library

    Native implementation of several mature differential privacy mechanisms and algorithms, suitable for consumption from software written in C, C++, Python, R, and other languages.

    Data Access SDK

    Data access layer supporting differentially private SQL against various engines, including Spark, SQL Server, Postgres, Presto, and CSV files

    Execution Service

    A reference implementation of an asynchronous service that can compose multiple different types of request over the same data source, while tracking total privacy spend.

    Stochastic Evaluator

    A testing harness that can stress-test any prospective differential privacy algorithm as a “black box” and find violations of privacy.
